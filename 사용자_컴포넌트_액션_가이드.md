# 사용자 컴포넌트 / 액션 가이드

이 문서는 `KirbyGameDll`에 사용자 컴포넌트와 사용자 액션을 추가하는 기본 방법을 정리한 문서입니다.

현재 프로젝트에서는 게임별 확장 코드를 `KirbyGameDll`에 넣는 것을 기본으로 합니다.

## 1. 사용자 컴포넌트 위치

보통 사용자 컴포넌트는 아래 폴더에 둡니다.

```text
KirbyGameDll/UserComponents/Scripts
```

등록은 아래 파일에서 합니다.

```text
KirbyGameDll/UserComponents/GameComponentRegistry.cpp
```

## 2. 사용자 컴포넌트 기본 형태

사용자 컴포넌트는 `Component`를 상속해서 만듭니다.

기본적으로 자주 구현하는 함수는 아래입니다.

- `Initialize()`
- `Release()`
- `Start()`
- `Update()`
- `GetInspectorName()`
- `GetSerializableType()`

필요할 때만 추가로 구현하면 되는 함수:

- `DrawInspector()`
- `Serialize()`
- `Deserialize()`
- `ResolveReferences()`
- `RegisterReferenceFields()`

## 3. 꼭 맞춰야 하는 것

### `GetSerializableType()`

이 값은 SceneData에 저장되는 component type 문자열입니다.
등록 이름과 같은 값으로 맞추는 쪽이 안전합니다.

### `GetInspectorName()`

Inspector에 보일 이름입니다.
사람이 읽기 쉬운 이름으로 정하면 됩니다.

## 4. 등록 방법

`GameComponentRegistry.cpp`에서 등록합니다.

현재 프로젝트는 `REGISTER_GAME_COMPONENT(...)` 매크로를 사용하고 있습니다.

예:

```cpp
REGISTER_GAME_COMPONENT(factory, SampleSpinComponent, "Sample Spin", "User");
```

여기서 보통 의미는 아래 정도로 보면 됩니다.

- 실제 C++ 타입
- 에디터에 보일 이름
- 카테고리 이름

## 5. Inspector를 그리고 싶을 때

`DrawInspector()`를 구현하면 됩니다.

예를 들어:

- 체크박스
- 숫자 입력
- 버튼
- 디버그 텍스트

같은 것을 넣을 수 있습니다.

주의:

- `KirbyGameDll` 쪽 inspector UI에서는 ImGui context 동기화를 위해
  `SyncImGuiContextForCurrentModule();`
  호출이 필요한 경우가 있습니다.
- 현재 샘플 사용자 컴포넌트들도 이 방식을 사용하고 있습니다.

## 6. GameObject / Component 참조를 inspector에 붙이고 싶을 때

지금은 `ReferenceFieldRegistry`를 쓸 수 있습니다.

예:

```cpp
void MyComponent::RegisterReferenceFields(ReferenceFieldRegistry& registry)
{
    registry.RegisterGameObjectRef("Target Object", &m_targetObject);
    registry.RegisterComponentRef("Target Rigidbody", &m_targetComponent, "Rigidbody2D");
}
```

이렇게 등록하면 inspector에서 공통 방식으로 그려집니다.

현재 지원되는 흐름:

- `GameObject` 참조 필드
- `Component` 참조 필드
- hierarchy에서 `GameObject` drag & drop
- 타입이 맞는 component 자동 찾기
- SceneData 저장/로드

## 7. 저장/로드를 하고 싶을 때

### 기본 데이터

`Serialize()` / `Deserialize()`에서 직접 저장합니다.

예:

- bool
- int
- float
- string
- 직접 관리하는 object id

### registry 기반 reference field

이건 이제 공통 저장/로드 기반이 있습니다.

즉,
`RegisterReferenceFields(...)`에 등록한 필드는
SceneData의 `referenceFields` 아래에 자동으로 저장/복원됩니다.

단,
기존에 따로 수동 저장하던 field는 그대로 직접 관리해도 됩니다.

## 8. 사용자 액션 위치

사용자 액션은 보통 아래 폴더에 둡니다.

```text
KirbyGameDll/UserActions/Scripts
```

등록은 아래 파일에서 합니다.

```text
KirbyGameDll/UserActions/GameActionRegistry.cpp
```

## 9. 사용자 액션 등록 방식

현재 버튼 action은 callback 본체를 SceneData에 저장하지 않습니다.
대신 `actionKey`만 저장하고, 시작할 때 registry에서 다시 연결합니다.

예:

```cpp
UIActionRegistry::RegisterAction(
    "SampleButtonAction",
    []() { ExecuteSampleButtonAction(); });
```

즉,
- SceneData에는 문자열 key 저장
- 실제 동작은 startup 때 다시 등록

구조로 이해하면 됩니다.

## 10. 작업 후 주의할 점

현재 사용자 C++ 컴포넌트 / 액션은 hot reload를 지원하지 않습니다.

즉,
- C++ 코드 수정
- DLL 다시 빌드
- 에디터 재실행

흐름으로 보는 편이 안전합니다.

으로 이해하면 됩니다.
