# 사용자 액션 추가 체크리스트

이 문서는 `KirbyGameDll`에 새 사용자 액션을 추가할 때 꼭 확인할 항목만 짧게 정리한 문서입니다.

## 1. 파일 만들기

새 액션 파일은 아래 폴더에 만듭니다.

- `Solution_Kirby/KirbyGameDll/UserActions/Scripts/`

보통 2개 파일을 같이 만듭니다.

- `MyAction.h`
- `MyAction.cpp`

## 2. 액션 함수 만들기

버튼에서 실행할 실제 함수를 만듭니다.

예시:

```cpp
void ExecuteMyAction();
```

```cpp
void ExecuteMyAction()
{
    OutputDebugStringA("MyAction executed.\n");
}
```

## 3. vcxproj에 파일 추가

VSCode에서 파일만 추가하면 빌드 대상에 자동으로 안 들어갈 수 있습니다.

그래서 아래 파일도 같이 수정해야 합니다.

- `KirbyGameDll/KirbyGameDll.vcxproj`
- 필요하면 `KirbyGameDll/KirbyGameDll.vcxproj.filters`

## 4. 액션 등록

액션 등록은 아래 파일에서 합니다.

- `KirbyGameDll/UserActions/GameActionRegistry.cpp`

예시:

```cpp
UIActionRegistry::RegisterAction("MyAction", []() { ExecuteMyAction(); });
```

여기서 문자열 키가 실제 버튼과 연결되는 이름입니다.

## 5. 버튼과 연결

UI 버튼은 SceneData에 callback 자체를 저장하지 않고,
`actionKey` 문자열만 저장합니다.

즉 연결 구조는 이렇게 생각하면 됩니다.

1. 버튼은 `actionKey`만 기억
2. 시작할 때 `UIActionRegistry`에서 같은 key를 찾음
3. 찾은 함수를 버튼에 연결

## 6. 이름 맞추기

아래 값은 정확히 같아야 합니다.

- `GameActionRegistry.cpp`에 등록한 key
- UIButton 쪽에서 쓰는 `actionKey`

하나라도 다르면 버튼을 눌러도 동작하지 않습니다.

## 7. 빌드 후 확인

새 C++ 액션을 추가했다면 보통 아래 순서로 확인합니다.

1. `KirbyGameDll` 빌드
2. `EditorApp` 실행
3. 버튼의 `actionKey`를 새 이름으로 설정
4. 실행 중 버튼을 눌러 동작하는지 확인

## 8. 자주 놓치는 부분

- 파일만 만들고 `KirbyGameDll.vcxproj`에 안 넣는 경우
- `GameActionRegistry.cpp` 등록을 빼먹는 경우
- 버튼 `actionKey`와 등록 문자열이 다른 경우
- DLL만 다시 빌드하고 에디터를 다시 안 켜서 예전 상태가 남는 경우

## 9. 주의할 점

지금 구조에서는 C++ 액션을 추가한 뒤 보통
DLL 재빌드와 에디터 재실행이 같이 필요합니다.

즉, 실행 중에 액션 파일만 바꿨다고 바로 반영되지는 않습니다.

## 한 줄 정리

사용자 액션 추가는
`파일 생성 -> vcxproj 추가 -> registry 등록 -> actionKey 연결`
순서로 생각하면 가장 쉽습니다.
