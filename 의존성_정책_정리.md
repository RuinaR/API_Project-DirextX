# 의존성 정책 정리

이 문서는 현재 프로젝트에서 어떤 프로젝트가 무엇을 참조하고 있는지,
그리고 앞으로 어떤 방향으로 정리하려고 하는지를 쉽게 정리한 문서입니다.

## 1. 현재 큰 구조

현재 솔루션의 큰 프로젝트는 아래와 같습니다.

- `EditorApp`
- `GameApp`
- `EngineFrameworkDll`
- `EngineEditor`
- `KirbyGameDll`

## 2. 현재 의존 방향

### EditorApp

- `EngineFrameworkDll`
- `EngineEditor`
- 실행 중 `KirbyGameDll.dll` 로드

즉, 에디터 실행기는
엔진 공통 기능 + editor 기능 + 게임별 확장을 같이 사용합니다.

### GameApp

- `EngineFrameworkDll`
- 실행 중 `KirbyGameDll.dll` 로드

즉, 게임 실행기는
editor 기능 없이 게임 실행에 필요한 쪽만 사용합니다.

### KirbyGameDll

- `EngineFrameworkDll`
- `EngineEditor`는 직접 참조하지 않는 방향 유지

즉, 게임별 DLL은
가능하면 runtime 쪽 공통 기능만 의존하는 것이 목표입니다.

### EngineEditor

- `EngineFrameworkDll`

즉, editor 기능은 엔진 공통 기능을 바탕으로 동작합니다.

## 3. 왜 이런 방향을 쓰는가

이 구조를 쓰는 이유는 간단합니다.

- `EditorApp`은 editor 기능이 꼭 필요함
- `GameApp`은 editor 기능이 없어야 더 깔끔함
- `KirbyGameDll`은 게임별 코드에 집중해야 함

즉,
"게임 실행에 꼭 필요하지 않은 editor 코드가 game 쪽으로 퍼지지 않게 하자"가 기본 생각입니다.

## 4. 현재 아직 남아 있는 혼합 지점

아직 완전히 분리된 것은 아닙니다.

특히 아래 파일들은 runtime/editor 책임이 일부 섞여 있습니다.

- `MainFrame`
- `WindowFrame`
- `RenderManager`

그래서 지금은 프로젝트 이름과 의존 방향은 더 정리됐지만,
내부 클래스 책임 분리는 아직 진행 중이라고 보는 편이 맞습니다.

## 5. 현재 기준으로 지키고 싶은 규칙

### 좋다고 보는 방향

- `EditorApp` -> `EngineFrameworkDll`, `EngineEditor`
- `GameApp` -> `EngineFrameworkDll`
- `KirbyGameDll` -> `EngineFrameworkDll`
- `EngineEditor` -> `EngineFrameworkDll`

### 피하고 싶은 방향

- `GameApp` -> `EngineEditor`
- `KirbyGameDll` -> `EngineEditor`
- runtime core가 editor 헤더를 직접 include하는 구조

## 6. 최근 정리된 부분

최근에 아래 직접 의존을 줄였습니다.

- `WindowFrame -> EditorSceneWorkflow` 직접 호출 제거
- `RenderManager -> EditorRenderFacade` 직접 호출 제거

이 둘은 callback 등록 방식으로 바뀌었습니다.
즉, runtime 쪽은 editor 코드를 직접 몰라도 되게 정리한 것입니다.

## 7. 왜 EngineFrameworkDll을 DLL로 바꿨는가

예전 `EngineFramework`는 static library였습니다.
그런데 이 상태에서는 EXE와 DLL이 각자 엔진 싱글턴을 따로 볼 수 있었습니다.

예를 들어:

- EXE에서 `MainFrame::Create()`
- DLL에서 `MainFrame::GetInstance()->Update()`

이렇게 되면 같은 전역처럼 보여도 실제로는 다른 인스턴스를 볼 수 있습니다.
그래서 실행 흐름이 꼬이거나 null 접근이 날 수 있었습니다.

이 문제를 줄이기 위해 공통 엔진을 `EngineFrameworkDll`로 정리했습니다.

## 8. 지금 단계에서 한 줄 정리

현재 의존성 정책은
"game 실행 경로에는 editor 의존을 최대한 넣지 않고,
editor 기능은 따로 모아 가는 방향"으로 이해하면 됩니다.
