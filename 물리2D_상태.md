# 물리2D 상태

이 문서는 현재 `KirbyEngine`의 2D 물리 시스템이 어느 정도까지 구현되어 있고,
어떤 제약이 있는지 쉽게 정리한 문서입니다.

## 1. 현재 사용 중인 물리 구조

현재 물리 시스템은 2D 전용 기준으로 정리되어 있습니다.

주요 타입:

- `Physics2D`
- `Rigidbody2D`
- `Collider2D`
- `BoxCollider2D`
- `CircleCollider2D`

기반 라이브러리는 `Box2D`입니다.

## 2. 현재 동작 방식

### Rigidbody2D가 있을 때

- `Rigidbody2D`가 body를 관리합니다.
- 같은 GameObject에 붙은 collider는 그 body의 fixture로 연결됩니다.

### Rigidbody2D가 없을 때

- collider가 static fallback body를 직접 만듭니다.
- 간단한 고정 오브젝트를 빠르게 만들 때는 편합니다.

## 3. 현재 지원하는 것

- Box collider
- Circle collider
- Trigger collider
- Collision / Trigger Enter, Stay, Exit
- Raycast
- active on/off에 따른 물리 활성/비활성 반영

## 4. 충돌 이벤트 규칙

### Collision 계열

- 둘 다 일반 collider일 때 사용합니다.
- `CollisionEnter`
- `CollisionStay`
- `CollisionExit`

### Trigger 계열

- 둘 중 하나라도 trigger면 trigger 이벤트로 처리합니다.
- `TriggerEnter`
- `TriggerStay`
- `TriggerExit`

### 주의할 점

삭제 예약이나 강제 종료 상황에서는 아래처럼 `nullptr`가 들어올 수 있습니다.

- `OnCollisionExit(nullptr)`
- `OnTriggerExit(nullptr)`

이건 비정상이라기보다,
"상대가 이미 정리 중이라 포인터를 안전하게 줄 수 없는 상황"을 허용한 정책입니다.

## 5. Raycast

`Physics2D::Raycast`는 현재 주요 2D collider를 대상으로 동작합니다.

- `BoxCollider2D`
- `CircleCollider2D`

에디터와 런타임에서 마우스 선택, hover 판정 등에 활용합니다.

## 6. 현재 제약

### fallback body 구조

`Rigidbody2D` 없이 collider만 여러 개 붙이는 방식은 가능은 하지만 권장하지 않습니다.

이유:

- collider마다 body가 따로 생길 수 있음
- transform sync가 복잡해짐
- 나중에 rigidbody를 붙였다 떼는 경로가 헷갈릴 수 있음

그래서 collider가 여러 개 필요한 오브젝트는
가능하면 `Rigidbody2D`를 같이 붙이는 쪽이 더 안정적입니다.

### 3D 물리 아님

현재는 2D 물리 전용입니다.
3D physics 시스템은 아직 구현되어 있지 않습니다.
